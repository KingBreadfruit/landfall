# Landfall

**Emergency logistics dispatch for disaster relief in Jamaica.**

Built for the Ackee Circle Hackathon 2026.

---

## Overview

When a hurricane makes landfall, the gap between chaos and coordinated relief comes down to logistics: knowing what is needed, where it is needed, and getting it there fast. Landfall is a lightweight dispatch tool that helps relief coordinators intake requests, match them to available resources, and track deliveries across affected communities — even on a shaky connection.

## The problem

In the hours and days after a disaster, coordination usually happens over scattered phone calls, WhatsApp groups, and paper lists. Requests get lost, supplies get double-shipped to one shelter while another goes without, and no one has a single view of what is happening on the ground.

## What Landfall does

- **Intake & triage** — capture relief requests (supplies, medical, evacuation) with location and urgency.
- **Resource matching** — pair incoming needs with available supplies, vehicles, and volunteers.
- **Dispatch & tracking** — assign runs and follow their status from queued to delivered.
- **Offline-first** — built as a Progressive Web App so coordinators can keep working when the network drops.
- **Shared situational view** — one live picture of open needs and in-progress deliveries.

## Tech stack

- **React** + **TypeScript**
- **Vite** build tooling
- **Progressive Web App** (installable, offline-capable)
- Tailwind CSS for styling

## Getting started

```bash
git clone https://github.com/KingBreadfruit/landfall.git
cd landfall
npm install
npm run dev
```

Then open the local URL printed in your terminal.
